

## Ключевые особенности тестов:

### 1. **Изоляция с `setUp`**
   - Каждый тест создает новую чистую библиотеку, что предотвращает влияние тестов друг на друга.

### 2. **Полное покрытие методов**
   - **add_book**: все ветки (успешное добавление, пустые поля, None, дубликаты ISBN, пробелы, типы данных)
   - **remove_book**: успешное удаление, несуществующая книга, типы ISBN
   - **find_book_by_title**: поиск одной книги, нескольких книг, несуществующей, регистрозависимость
   - **list_books**: проверка копирования, сохранение порядка, пустая библиотека

### 3. **Проверка исключений**
   - Использование `assertRaises` для проверки всех случаев выброса `ValueError`
   - Проверка текста сообщений об ошибках

### 4. **Краевые случаи**
   - Поля с пробелами (`"   "`)
   - Разные типы данных для ISBN (строка vs число)
   - Работа с пустой библиотекой
   - Изменение возвращенного списка не влияет на внутреннее состояние

### 5. **Комплексные тесты**
   - Полный рабочий процесс, объединяющий все методы

## Запуск тестов:

```bash
python test_library.py
```

или с подробным выводом:
```bash
python -m unittest -v test_library.py
```

Все тесты должны пройти успешно, демонстрируя корректную работу класса `Library`.
